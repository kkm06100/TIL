## inode는 파일의 메타데이터 객체입니다.

`df -i`
명령어를 통해
```
❯ df -i
파일 시스템             Inodes   IUsed    IFree IUse% 마운트위치
tmpfs                  2008418    2448  2005970    1% /run
/dev/nvme0n1p2        31195136 2529417 28665719    9% /
tmpfs                  2008418      82  2008336    1% /dev/shm
tmpfs                  2008418       4  2008414    1% /run/lock
efivarfs                     0       0        0     - /sys/firmware/efi/efivars
/dev/nvme0n1p1               0       0        0     - /boot/efi
tmpfs                   401683     232   401451    1% /run/user/1000
tmpfs                  2008418       1  2008417    1% /run/qemu
```

해당 디스크에서 남은 inode의 양을 볼 수 있습니다.

* inode의 세부 정보에 대해서 살펴봤습니다.
```
❯ stat tbl_meal.csv
  파일: tbl_meal.csv
  크기: 135214    	블록: 272        입출력 블록: 4096   일반 파일
장치: 259,2	아이노드: 10761305    링크: 1
접근: (0664/-rw-rw-r--)  UID: ( 1000/kkm06100)   GID: ( 1000/kkm06100)
접근: 2026-04-14 20:05:42.319284570 +0900
수정: 2025-12-02 22:01:55.994738100 +0900
변경: 2025-12-02 22:01:55.994738100 +0900
생성: 2025-12-02 22:01:55.878734134 +0900

```
그리고 inode에는 해당 정보가 포함되어 있었습니다.
  - 일반 파일
  - 크기 135214 바이트
  - inode 번호 10761305
  - 링크 수 1
  - 소유자 kkm06100:kkm06100
  - 권한 0664 (-rw-rw-r--)

### 회고
* inode가 파일 시스템에서 메타데이터 역할을 어떻게 하느지 알았습니다.
* 특히 inode의 최대 개수가 디스크별로 정해져있다는 것을 처음 알게 되었습니다.
